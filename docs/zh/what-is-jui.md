# JUI 是什么?

JUI 是一个基于原生 JavaScript 的组件库，意在帮助初学者快速学习和构建网页而设计。

1. 帮助初学者在学习过程中获得及时的结果反馈，逐步增加自我认同，获得进一步学习的动力。
2. 帮助初学者在学习过程中，通过实际操作和实践，逐步建立起对 Web UI 的理解和应用能力。

## 设计原则

JUI 的定义完全遵循 W3C 的 Web 准则，用规范和简洁的语法定义基础结构，留下充足的用户个性化空间。

## 组件成员

JUI 包含 `CSS` 层叠样式表和 `JavaScript` 组件库，前者定义组件的样式，后者定义组件的行为。

1. 基础原子类：预设的常用原子类，帮助快速定义元素样式。
2. 组件：默认提供多种常用组件，帮助快速定义交互行为。

## 组件文件

JUI 提供三种不同形式的 `JavaScript` 文件，按需使用：

::: details `jui.umd.js` UMD 规范

载入后立即加载执行。全量组件挂载于 `jui` 对象。

:::

::: details `jui.esm.js` ES Module 标准

ES Module 标准。全量组件挂载于 `jui` 对象，不支持 Tree-shaking。

:::

::: details `jui.module.js` ES Module 标准

ES Module 标准，支持 Tree-shaking，有效减小打包体积。

:::

JUI 提供两种不同形式的 `CSS` 文件，按需使用：

- `jui.min.css` 全量 `CSS` 样式文件。
- `jui.pure.min.css` 不包含工具类的、纯净的主题和组件 `CSS` 样式文件。

## 命名规范

- 基础规则：
  1. 使用驼峰式命名法，首字母小写，后续每个单词的首字母大写。
  2. 内部方法使用 `_` 前缀，并对方法名和属性名进行压缩打包。
  3. 外层容器按品牌命名，组件对象按业务命名，内部成员按成员命名。
- 命名原则：
  1. 方便用户自定义
  2. 兼容主流 CSS 工具(Tailwind CSS)
  3. 尽量不污染用户业务特征
- CSS 类样式命名
  1. 组件类：仅外层容器使用 `j-` 前缀命名，如: `j-modal`；内部子元素类无前缀，如: `modal-header`。
  2. 状态类：使用 `is-` 前缀命名，如: `is-active`。
  3. 工具类：继承 `tailwinds CSS` 命名规范。
- JavaScript 组件命名
  1. 组件：与 CSS 类样式命名一致，如: `modal`。
  2. 方法：使用驼峰式命名法，首字母小写，后续每个单词的首字母大写，如: `modal`、`showLoading()`。
  3. 事件：使用 `on` 前缀命名，如: `onShow`。事件状态对应一般时态和完成时态，如 `onShow` 和 `onShown`。
